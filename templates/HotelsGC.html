Buscador de hoteles (Gran Canaria)

Automatización de captura de datos + dashboard en Power BI para explorar precios, reputación y tendencias.

Python Selenium Pandas Azure SQL Power BI
Contexto

Comparar precios y calidad de alojamientos sin revisar portales manualmente (y sin perder histórico).

Qué construí

Un pipeline que captura datos, los normaliza, los guarda en base de datos y alimenta un reporte interactivo en Power BI.

Valor aportado
  • Visión semanal de tendencias de precio y puntuación.
  • Comparativas por zona, tipo de habitación y fechas.
  • Menos trabajo manual y mejor trazabilidad del histórico.

Dashboard (Power BI)

Para una mejor experiencia, visualízalo en sobremesa o tablet.

Cómo funciona

1) Captura

Scraping automatizado (Selenium) con parámetros definidos (fechas, zona, huéspedes).

2) Limpieza

Normalización con Python/Pandas y cálculo de métricas (p. ej. precio por noche).

3) Persistencia

Carga a base de datos (Azure SQL) para conservar histórico y consulta eficiente.

4) Consumo

Reporte en Power BI conectado a la base de datos para exploración y comparativas.

  • Frecuencia típica: ejecución semanal.
  • Datos capturados: nombre, ubicación, tipo de habitación, precio, opiniones y puntuación.
  • Objetivo de diseño: minimizar trabajo manual y mantener histórico para análisis de tendencias.
